# Retrospective — Loops 0–3 (Retro 1)

_Date: 2026-06-27 · Facilitator: Project Manager · Present: all roles_

First studio retro. Blameless: improve the system, not the person. ~1.5 hours of delivery,
~5 releases, the studio framework itself grew mid-stream. Honest look back, together.

## What we shipped
- **Loop 0** — Playable v0: three.js ocean with GPU Gerstner swell, arcade sloop sailing,
  scattered islands, follow camera, HUD. Plus the CI/CD release pipeline (headless Puppeteer
  playtest gate → Pages deploy → `v0.0.YYYYMMDDHHmmSS` datetime tags), the studio framework,
  and the GitHub backlog (9 epics #1–9, slices #10–23).
- **Loop 1** — Bow wake + trailing foam that scales with speed (#18). QA caught the sail
  rendering invisible and filed #23.
- **Framework expansion** (owner-driven) — Extracted a pure physics module + TDD harness
  (`node:test`), added a unit-test CI gate; added **Sound Engineer + Musician** agents; stood up
  the QA quality system (`CHECKLIST.md`, `RUBRIC.md`, gallery, multi-screenshot evaluation);
  wrote the parallel-dev + issue-hygiene protocol (`PARALLEL.md`, `in-progress` label).
- **Loop 2** (first parallel batch) — Visible billowing sail + rigging (#23) **and** procedural
  sea/wind/gull ambience with a mute toggle (#14), built on disjoint files, merged serially.
- **Loop 3** — Wind compass + point-of-sail label, with TDD'd pure `pointOfSail()` /
  `relativeWindAngle()` (#10).
- **Totals:** ~33 unit tests, a green headless playtest gate, ~5 public releases. Live at
  https://cakuki.github.io/tidewake/.

## What worked (keep)
- **Always-shippable tiny slices.** Every loop ended with a playable, released build. Momentum
  never stalled; `main` stayed green.
- **Two CI gates earn their keep.** The headless playtest gate (renders + sails + zero console
  errors) plus the unit-test gate caught regressions cheaply and made releases boring.
- **TDD for pure logic.** Physics and `pointOfSail()` were specified by failing tests first —
  fast, confident, and the right place to invest (we don't TDD shaders or feel).
- **Parallel disjoint-file dev.** Sail (#23) + audio (#14) shipped in one batch with no
  collisions because the Tech Lead carved non-overlapping ownership and the PM serialised merges.
- **QA screenshot review caught the invisible sail.** A pure functional gate would have passed
  it. Human-eyes-on-pixels is a real gate, not a nicety.
- **Procedural / no-asset approach.** Sea, audio, and wake are all generated — the build stays
  tiny, openable, and instant to deploy. No asset pipeline tax yet.

## What didn't (improve)
- **`main.js` is becoming a contention hotspot.** Every feature that needs wiring touches it,
  which quietly serialises the very work `PARALLEL.md` exists to parallelise. This is the #1
  flow risk going forward. → filed **#24** (modularise into `src/systems/`).
- **Headless swiftshader can't validate visuals.** The CI renderer draws the 3D scene dark, so
  the gate can't see what the player sees (it missed the invisible sail; QA's real browser caught
  it). Visual confidence depends entirely on the manual Chrome-MCP pass — we should make that
  habit explicit and comparative, not ad hoc.
- **Node-20 GitHub-Actions deprecation annotation.** The workflow rides actions on the
  deprecated Node-20 runtime; harmless today, a hard failure later. Cheap to fix now, costly to
  hit mid-flow.
- **No save/load yet.** A refresh wipes the player's position/heading (#11). As soon as there's
  anything to *do*, losing progress will sting.
- **Art is still primitive.** The "ship" is boxes (#13). Fine for prototyping sailing feel, but
  it caps the charm half of our "realistic world, funny people" tone.
- **No gameplay verbs.** We have beautiful sailing and *nothing to sail toward*. Trade, ports,
  combat, reputation — none are playable. The north-star fantasy ("rise to pirate or governor")
  is **not yet a game**, it's a sailing toy. This is the biggest product gap.

## Experiments to try
- **If** we split `main.js` into a tiny `src/systems/` registry, **then** the next parallel
  batch ships with zero shared-file conflicts (measure: merge conflicts per batch).
- **If** QA archives one gallery shot per release and diffs against the previous, **then** visual
  regressions get caught before players do (measure: regressions caught pre-release).
- **If** we ship one real **gameplay verb** (dockable port → simple trade) before more polish,
  **then** playtesters describe a *goal*, not just "nice water" (measure: can a fresh player state
  an objective?).
- **If** we drop in a single glTF sloop behind the existing ship seam, **then** charm jumps with
  no architecture churn (measure: QA charm-rubric score vs. previous release).
