## Studio / process review

**What's reliable now (keep).**
- **Retro 7's hardening held — no regressions.** No 0-tool-use / injected-"output-style" glitch; clean
  trees throughout; the hard ritual trigger fired this very retro on schedule (counter hit 4 → this
  subagent is the next dispatch). The guardrails keep working.
- **Self-shipping cycle-runners stayed clean** — 3 green slices, gate green throughout, zero escaped
  prod bugs, each runner self-committed named paths, closed its issue, and appended its own loop-log
  row. The lean orchestrator protocol is humming.
- **TDD-first held on every slice** — slide (+16), islands (+12), ballad (+16) all landed
  test-first; the +56 tests this block were genuine coverage, not afterthoughts.

**What hurt / the findings.**
1. **The retro cadence is too frequent for this fast loop (Mandate 1).** Retros 6, 7, and 8 all ran
   within ~14 cycles. At "every 3–4 loops" with self-paced cycles minutes apart, the retro becomes
   low-value ceremony that competes with shipping — each one re-reads the whole runbook, re-derives
   context, and produces ever-smaller deltas (this block had essentially no new process friction; the
   hardening from Retros 5–7 is holding). The *spirit* of a scheduled, never-deferred ritual is right;
   the *frequency* is wrong for a minutes-apart loop. **Fix:** relax the cadence to **every ~7–8
   cycles** and lift the HARD-trigger threshold from 4 to **7** to match. The deep-learning loop stays
   every ~10. (Applied this retro to `LOOP.md` §1, §4, the HARD-trigger section, and the Changelog.)
2. **The release-trigger / release-policy mismatch (#89) was still open tech debt (Mandate 2).**
   `release.yml` used `paths-ignore` (`studio/**`, `docs/**`, `**/*.md`), so a push touching
   `scripts/**`, `tests/**`, `.github/**`, `package*.json`, or `manifest.webmanifest` *did* trigger a
   release despite the stated "src/ + index.html only" policy — we'd been using `[skip ci]` as a manual
   workaround (Retro 7 flagged it; the script commit that retro used `[skip ci]`). **Fix:** switched the
   push trigger to an **allow-list** `paths: ['src/**', 'index.html']` — the policy is now the
   mechanism. Tooling/doc/test commits no longer release and no longer need `[skip ci]`. Editing
   `release.yml` itself also won't release (not in the allow-list; use `workflow_dispatch`). YAML
   validated. **Caveat noted on the workflow + referenced to #38:** unit tests currently run *only*
   inside the release job, so script/test-only changes are now un-CI-checked on push — the proper home
   for that coverage is the **#38 PR-validation gate** (not built here). #89 closed.
