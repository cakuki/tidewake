# Retrospective — Loop <N> (or Loops <N–M>)

_Date: YYYY-MM-DD · Facilitator: Project Manager · Present: all roles_

Held every 3–4 loops. Blameless: improve the system, not the person. Copy this file to
`retros/loop-<N>.md` and fill it in. Action items must name an owner **and** the exact
agent definition / runbook / file to update — otherwise nothing changes.

## What we shipped
- Releases, features, and player-facing changes since the last retro.

## What worked (keep)
- Practices, tools, and decisions to deliberately continue.

## What didn't (improve)
- Friction, rework, missed acceptance, off-tone work, or flow stalls — with the why.

## Experiments to try
- Small, reversible changes to test next block. One hypothesis each: "If we …, then …".
